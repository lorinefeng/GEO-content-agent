# GEO Content Agent Skill

本技能封装了GEO内容投放Agent的完整工作流程，包括商品数据获取、内容生成和自动化发布。

## 概述

GEO（Generative Engine Optimization）内容投放的目标是让AI大模型"爱上"你的SKU，通过在互联网投放针对性内容，提升品牌在AI回答中的曝光率。

### 核心能力

1. **商品数据获取** - 从电商平台API获取商品信息
2. **竞品信息搜索** - 联网分析市场竞品情况
3. **双策略内容生成** - 评测对比型 + 用户画像匹配型
4. **平台风格适配** - 根据目标平台特征优化内容
5. **自动化发布** - 通过浏览器自动化发布内容

## 一、商品数据获取

### Zara API 调用

```python
from agents.fetch_zara_data import ZaraAPI, fetch_zara_products

# 初始化API
api = ZaraAPI()

# 搜索商品
products = fetch_zara_products(
    category="女士",           # 品类：女士/男士/儿童/家居
    keywords=["春季新款", "外套", "连衣裙"],
    limit_per_keyword=3
)
```

### API 端点

| 接口 | 说明 | 返回数据 |
|-----|------|---------|
| `search_products()` | 搜索商品 | 商品列表（名称、价格、图片、标签） |
| `get_product_details()` | 商品详情 | 材质、规格等详细信息 |
| `get_tag_info()` | AI标签 | 主品类、白名单标签等 |

### 关键字段映射

API返回使用 `rows` 而非 `products`，字段名映射：
- `productName` → 商品名称
- `mainImage` → 商品图片
- `categories` → 品类列表
- `tags` → 标签列表（用于用户画像推理）

## 二、内容生成策略

### 策略一：评测对比型（DeepSeek偏好）

**目标用户**：追求详细技术参数和客观对比的理性消费者

**内容特点**：
- 包含详细规格对比表格
- 材质工艺技术解析
- 竞品横向对比（优衣库、H&M、Massimo Dutti等）
- 客观优缺点分析
- FAQ问答模块

**模板结构**：
```markdown
# [商品名称] 深度评测与竞品对比

## 一、核心规格对比表
| 参数 | 本品 | 竞品A | 竞品B |
|-----|------|-------|-------|

## 二、材质与工艺解析
...

## 三、优缺点分析
### ✅ 核心优势
### ❌ 主要不足

## 四、购买建议
## 五、常见问题FAQ
```

### 策略二：用户画像匹配型（干货内容）

**目标用户**：特定用户群体（如都市通勤女性）

**内容特点**：
- 第一人称讲述
- 场景化需求匹配
- 具体搭配方案
- 温暖亲切的语气
- 适当使用emoji

**模板结构**：
```markdown
# 🌸[场景词]｜[价格][商品]推荐

## 为什么这件能解决你的[痛点]？
...

## 搭配方案（附场景说明）
| 场景 | 搭配公式 | 效果 |
|-----|---------|------|

## 值不值得买？
```

## 三、什么值得买平台适配

### 平台内容特征总结

**标题写作规则**：
1. **数字驱动**：必须包含具象数字（"4招"、"7个缺点"、"14天实测"）
2. **情绪词汇**：使用"别急"、"离谱"、"太坑了"、"血泪真相"
3. **利益导向**：直接点出核心收益（"省钱"、"值不值"）
4. **交互提问**：引导评论（"这买卖值吗？"）

**内容结构模式**：
- **攻略型**：痛点引入 → 核心论点(3-5点) → 案例数据 → 总结建议
- **评测型**：开箱外观 → 核心参数实测 → 使用场景 → 优缺点总结

**流量规律**：
- 极高信息密度
- 第一人称增强真实感
- 图文配比高（每200-300字配图）
- 精准分类标签

### 示例标题

```
❌ 不好的标题：Zara羊毛外套分享
✅ 推荐标题：Zara这件纯羊毛外套我穿了14天，5个血泪真相曝光！¥549值不值？
```

## 四、自动化发布流程

### 什么值得买发布要点

1. **发布入口**：顶部导航 "爆料投稿" → "写篇文章"
2. **URL**：`https://post.smzdm.com/tougao/`
3. **编辑器**：ProseMirror框架
4. **中文输入**：需通过JavaScript设置（键盘输入中文会失败）

### 关键代码片段

```javascript
// 设置标题
const titleField = document.querySelector('.article-title');
titleField.value = '标题内容';
titleField.dispatchEvent(new Event('input', { bubbles: true }));

// 设置正文（ProseMirror编辑器）
const editor = document.querySelector('.ProseMirror');
editor.innerHTML = content.split('\n').map(line => 
  line.trim() ? `<p>${line}</p>` : '<p><br></p>'
).join('');
editor.dispatchEvent(new Event('input', { bubbles: true }));
```

### 草稿保存确认

- 编辑器底部显示"草稿已保存"绿色标识
- 用户中心 → 我的文章 → 草稿箱 确认

## 八、常见问题

### Q1: API调用失败？
检查Token是否过期，确认网络连接正常。

### Q2: 中文标题输入失败？
浏览器自动化时，使用JavaScript直接设置value，而非键盘输入。

### Q3: 编辑器内容丢失？
ProseMirror需要触发input事件才能同步状态，记得dispatch事件。

### Q4: 草稿没保存？
确认网络正常，检查编辑器底部是否显示"草稿已保存"。
